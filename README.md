# Proyecto 1

## Descripción
Proyecto 1. Incluye un archivo excel para apps web (UI, validación de campos, flujo de usuario).

### Objetivo
Aprender los fundamentos de calidad de software y QA mediante diseño de checklists y primeros reportes de defects.

---

## Actividades principales

1. Conceptos básicos: ¿Qué es QA? Diferencia entre QA y QC.
2. Tipos de pruebas: funcionales, no funcionales, smoke, regression, exploratorias.
3. Ciclo de vida del software y metodologías: Waterfall vs Agile/Scrum.
4. Introducción a bugs: severidad y prioridad.
5. Diseño de checklists básicos: para apps web simples (UI, validación de campos, flujo de usuario).

---

## Logros
- Creación de checklists en Excel.
- Primeros reportes de defects (documentados en el Excel).

---

## Estructura del proyecto

- → Contiene el archivo Proyecto1.xlsx

---

## Cómo usar
1. Clonar el repositorio
2. Abrir el Excel en 'docs'
3. Revisar y aplicar los checklists para pruebas manuales en apps web simples
